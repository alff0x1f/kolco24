# План: сортировка по текущему периоду на странице donate

## Context

На странице `/donate/` таблица участников сортируется по «последнему» периоду — то есть по периоду с максимальной датой
среди активных. Если в базе уже заведён будущий период (например, «Осень 2026» с датой 2026-07-01), сортировка идёт по
нему, хотя взносы за него ещё не собирались. Нужно сортировать по **текущему** периоду — последнему из тех, чья дата не
превышает сегодняшнюю.

## Изменяемые файлы

- `src/donate/views.py` — единственное место с логикой сортировки (`build_donor_table`)

## Реализация

В методе `build_donor_table()` заменить определение `latest_period_id` на логику «текущего периода»:

**До:**

```python
latest_period_id = periods[-1].id
```

**После:**

```python
today = timezone.now().date()
past_or_current = [p for p in periods if p.date <= today]
current_period = past_or_current[-1] if past_or_current else periods[0]
current_period_id = current_period.id
```

Добавить импорт `from django.utils import timezone`. Переменные `latest_period_id` / `paid_latest` переименованы в
`current_period_id` / `paid_current`.

## Поведение после изменения

- Если сегодня `2026-04-23`, а периоды: `Весна 2026 (2026-01-01)`, `Осень 2026 (2026-07-01)` — сортировка идёт по «Весна
  2026».
- Если все периоды в будущем — fallback на самый ранний.
- Визуальный класс строки (`dues-row--paid` / `dues-row--pending`) тоже определяется по текущему периоду.

## Проверка

```bash
.venv/bin/pytest src/donate/ -v
.venv/bin/ruff check src/donate/views.py
```

Вручную: добавить в базу будущий период (дата > сегодня) и убедиться, что таблица сортируется по предыдущему периоду.
