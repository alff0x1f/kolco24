# Mobile App Backend: HMAC-Signed Endpoints + Install Stats

## Overview
- New Django app `apps.mobile` serving endpoints for the mobile app (iOS/Android) **without user registration**.
- First protected endpoint: the **legend** (`Checkpoint.description` per race), accessible only to the app via required mobile headers + HMAC signature — not via plain `curl`. The shared secret is baked into the app binary; we accept it can be extracted (not 100% protection, by design).
- Collects usage statistics at **per-install granularity** (app-generated `install_id`), aggregated in an `AppInstall` row.
- Integrates as a self-contained app: existing `/api/` (timing/scanning) and `BearerTokenPermission` (export to another site) are **not touched**. Only additions: the new app, 1 line in `config/urls.py` (the `include` import already exists), 2 settings + reads, 1 line in `kolco24.env.example`.

## Solution Overview
- **Approach A (chosen in brainstorm):** a DRF `permission_class` does the crypto check (pure, only stashes parsed metadata on `request`); a base `AppAPIView` records the `AppInstall` stat in `initial()` *after* permissions pass. This keeps `has_permission` free of heavy side effects while guaranteeing stats are written only for verified requests.
- **Fail closed:** empty/missing `MOBILE_APP_SECRET` → every request 403, so a misconfigured deploy never leaks the legend.
- **Neutral errors:** any verification failure returns a single `403 {"detail": "Forbidden"}` — no hint about *which* check failed (don't help brute-forcing). `race_id` 404 is only reachable *after* a valid signature.
- **Replay mitigation:** signed `timestamp` with a ±300s freshness window. No nonce (deliberately out of scope).

## Technical Details

### Headers (request from app)
| Header | Content |
|---|---|
| `X-Install-Id` | UUID generated by the app on first launch, stored locally |
| `X-App-Platform` | `ios` / `android` |
| `X-App-Version` | e.g. `1.4.0` |
| `X-App-Ts` | unix time (seconds) at request build |
| `X-App-Sig` | hex of HMAC-SHA256 |

### Canonical string (what we sign)
```
method.upper() + "\n" + full_path + "\n" + ts + "\n" + sha256_hex(body)
```
- `full_path` = `request.get_full_path()` (path **with** query-string, so `?param=` can't be tampered).
- `sha256_hex(body)` = hex SHA-256 of the raw request body (empty body for GET → hash of `b""`). Signing the body hash (not the body) keeps the canonical small.

### `signing.py`
- `build_canonical(method: str, full_path: str, ts: str, body: bytes) -> str`
- `sign(secret: str, canonical: str) -> str` → `hmac.new(secret.encode(), canonical.encode(), sha256).hexdigest()`
- `verify(secret: str, canonical: str, provided_sig: str) -> bool` → `hmac.compare_digest(sign(...), provided_sig)` (constant-time).

### `permissions.py` — `SignedAppPermission(BasePermission).has_permission`
1. `secret = getattr(settings, "MOBILE_APP_SECRET", "") or ""` → empty → `False` (fail closed).
2. read `X-App-Sig`, `X-App-Ts`, `X-Install-Id`; any missing → `False`.
3. `ts` not int → `False`.
4. `abs(time.time() - ts_int) > settings.MOBILE_APP_TS_WINDOW` → `False`.
5. `canonical = build_canonical(request.method, request.get_full_path(), ts, request.body)`; `verify(...)` false → `False`.
6. success → stash `request.app_meta = {"install_id": install[:64], "platform": h.get("X-App-Platform","")[:16], "app_version": h.get("X-App-Version","")[:32], "ip": _client_ip(request)}` → `True`.
- helper `_client_ip(request)`: prefer first `X-Forwarded-For` entry, fall back to `REMOTE_ADDR`.

### `views.py`
- `AppAPIView(APIView)`: `authentication_classes = []`, `permission_classes = [SignedAppPermission]`.
  - override `initial(self, request, *args, **kwargs)`: call `super().initial(...)` (runs `check_permissions`) **then** `self._record_install(request)`.
  - `_record_install(request)`: read `request.app_meta` (skip if absent); `AppInstall.objects.update_or_create(install_id=..., defaults={platform, app_version, last_ip})`; then `AppInstall.objects.filter(install_id=...).update(request_count=F("request_count")+1)`. Wrap the whole thing in `try/except Exception` with `logger.exception(...)` — a stats write failure must **not** break the response.
- `LegendView(AppAPIView).get(request, race_id)`: `race = get_object_or_404(Race, pk=race_id)`; `qs = Checkpoint.objects.filter(race=race).order_by("number", "id")` (stable order — `number` can repeat across `cost` variants, so the `id` tie-breaker keeps output deterministic and the ordering test non-flaky); `Response({"race": race_id, "checkpoints": LegendCheckpointSerializer(qs, many=True).data})`.

### Constraints / accepted limitations (from plan review)
- **Body signing is GET-only for now.** `build_canonical` hashes the request body generically, but the only endpoint is GET (empty body → `sha256_hex(b"")`). DRF's `request.body` can raise `RawPostDataException` once a stream is parsed; before adding any body-bearing POST `AppAPIView`, revisit how the raw body is read in the permission. A unit test asserts a GET signs `sha256_hex(b"")`.
- **Stats are best-effort.** `_record_install` uses `update_or_create` + a separate `F()` increment (two non-transactional queries); under concurrent requests from the same `install_id` an insert race or lost increment is possible, and any `IntegrityError` is swallowed by the try/except. Exact, race-free counting is explicitly out of scope — the legend response must never fail because of stats.

### `serializers.py`
- `LegendCheckpointSerializer(ModelSerializer)` on `website.Checkpoint`, `fields = ["number", "cost", "type", "description"]` (never expose `id`/`iterator`/`year`).

### Model `AppInstall`
- `install_id` CharField(max_length=64, unique=True, db_index=True)
- `platform` CharField(max_length=16, blank=True)
- `app_version` CharField(max_length=32, blank=True)
- `first_seen` DateTimeField(auto_now_add=True)
- `last_seen` DateTimeField(auto_now=True)
- `last_ip` GenericIPAddressField(null=True, blank=True)
- `request_count` PositiveIntegerField(default=0)
- admin: registered read-only list (`list_display` install_id/platform/app_version/request_count/last_seen; all fields read-only) for viewing stats.

### Settings & URLs
- `src/config/settings.py`: `MOBILE_APP_SECRET = os.getenv("MOBILE_APP_SECRET", "")`; `MOBILE_APP_TS_WINDOW = 300`; add `"apps.mobile"` to `INSTALLED_APPS`.

- `apps/mobile/urls.py`: `app_name = "mobile"`; `path("race/<int:race_id>/legend/", LegendView.as_view(), name="legend")` → `GET /app/race/<int:race_id>/legend/`.
