# Mobile app: `teams` + `sync` endpoints

## Overview
Add two signed mobile-app endpoints under `src/apps/mobile`, per `src/apps/mobile/README.md`:

- `GET /app/race/<int:race_id>/teams/` (`name="mobile:teams"`) — full team list for a race
  with nested member composition, supporting conditional GET via `ETag`/`If-None-Match`.
- `GET /app/race/<int:race_id>/sync/` (`name="mobile:sync"`) — a **pure manifest**: per-resource
  version strings + handoff lease flags, no data serialization. Lets the client probe with one
  cheap signed request and fetch only changed resources.

The existing `GET /app/race/<id>/legend/` endpoint is **left untouched** (not deleted). It stays
unversioned (no `tags`, no ETag) until a later redo, so the manifest's `versions` carries **only
`teams`** for now.

Both endpoints are `AppAPIView` subclasses (HMAC signature + best-effort `AppInstall` stats),
reusing the existing signing/permission machinery. Self-contained: no changes to `/api/`,
`donate`, or `website` beyond one additive model field.

## Solution Overview
- **One fingerprint, two consumers.** `versioning.teams_version(race_id)` is the single source of
  truth. `TeamsView` wraps it in quotes for the strong `ETag` header; `SyncView` emits the bare
  value in `versions.teams`. Client compares the manifest value against the stored ETag
  (normalizing quotes) to decide whether to re-fetch `/teams/`.
- **Member edits must move the version.** `Athlet` gains `updated_at` (`auto_now`) so a rename
  shifts `MAX(Athlet.updated_at)` and thus the fingerprint — the README's `MAX(Team.updated_at)`
  alone would miss it.
- **sync is the cheap probe** (two aggregates, no serialization) and therefore has **no**
  `If-None-Match`/304 itself.
- **Lease stubbed.** `data_source` comes from `MOBILE_DATA_SOURCE` (default `"cloud"`); cloud →
  `lease_expires_at: null`. Per-race local-server lease computation is explicitly deferred.

## Technical Details
- **`Athlet.updated_at`**: `models.DateTimeField(auto_now=True)`. Migration `0076` adds it;
  existing rows get the migration timestamp (harmless — just seeds the first fingerprint).
- **`teams_version(race_id)`**:
  - `Team.objects.filter(category2__race_id=race_id).aggregate(Max("updated_at"), Count("id"))`
    (`TeamManager` already excludes `is_deleted`).
  - `Athlet.objects.filter(team__category2__race_id=race_id).aggregate(Max("updated_at"))`.
  - Fingerprint: `hashlib.blake2b(f"{max_team_updated}|{max_athlet_updated}|{team_count}".encode(), digest_size=8).hexdigest()`.
  - `None` aggregates render as the literal string `"None"` → stable, non-crashing for an empty race.
  - Returns **bare** hex (no quotes).
- **`TeamSerializer`** (mobile): fields `id`, `teamname`, `category2` (`serializers.IntegerField(source="category2_id")`), `ucount`, `paid_people`, `start_time`, `finish_time`, and nested `members` (`name`, `birth`, `number_in_team`). The `members` field iterates the **prefetched** relation directly (e.g. `obj.athlet_set.all()`) — it must NOT call `.order_by(...)` in the serializer (that re-queries and defeats the prefetch / N+1). Ordering is supplied by the view's `Prefetch` (below).
- **`TeamsView.get`**: `race = get_object_or_404(Race, pk=race_id)`; `etag = teams_version(race_id)`; `quoted = f'"{etag}"'`.
  - **304 path**: if `request.headers.get("If-None-Match") == quoted` → return Django's `HttpResponseNotModified()` (guaranteed empty body, idiomatic) with `resp["ETag"] = quoted` set manually, and **no serialization** (early return before touching the team queryset). Chosen over DRF `Response(status=304)` because the latter still runs rendering and the README requires an empty 304 body.
  - **200 path**: serialize `Team.objects.filter(category2__race=race).order_by("id").prefetch_related(Prefetch("athlet_set", queryset=Athlet.objects.order_by("number_in_team", "id")))`, set `resp["ETag"] = quoted`, return `{"race": race_id, "teams": [...]}`. `order_by("id")` gives deterministic team order for stable tests.
  - **Filtering note (intentional):** a race "owns" teams via `category2.race`, so teams with `category2=None` are out of scope — excluded from both the fingerprint and the list. This is a deliberate decision, not an oversight.
- **`SyncView.get`**: `get_object_or_404(Race, pk=race_id)`; return `{"race": race_id, "data_source": settings.MOBILE_DATA_SOURCE, "lease_expires_at": None, "versions": {"teams": teams_version(race_id)}}`.
- **settings**: `MOBILE_DATA_SOURCE = os.getenv("MOBILE_DATA_SOURCE", "cloud")` near `MOBILE_APP_SECRET`.
